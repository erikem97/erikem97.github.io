PROJECTS

Newton-Pepys-problem

The Newton–Pepys dice probablity problem is defined as:
Which is more likely?

  • Six fair dice are tossed independently and at least one “6” appears.
  • Twelve fair dice are tossed independently and at least two “6”s appear.
  • Eighteen fair dice are tossed independently and at least three “6”s appear.

This problem was solved in Python by simulating the problem one million times using the random and math libraries along with matplot lib for presentling the results.

Deathroll

Deathroll is a casion game in World of warcraft played by two bettors based rolling diffrent dices. In this game one of betters has to go first and roll a dice and if the number that shows up is one, that bettor loses. In this project I used Python to simulate the statistical advatange of being the bettor whom rolls their dice last.

This problem would require many months to simulate on a normal work PC so therefore I contacted Lysator, the Academic Computer Society at Linköping Univesity whom helped me simulate the problem on their super computers to shorten the ammount of time required tofinish the simulations. For this project I learnt and used multipule diffrent libraries such as pickle to be able to load and store data, argparse to easily simulate diffrent scenarios on the super computers, numpy to handle mathematical the process and subprocesses to dived the project into diffrent files for easier use. I also solved the issue analyticaly compared it the with simulated results.

Runescape bot

Old School Runescape is a MMO-RPG video game I used to play as a child in which you can build a character and level diffrent skills such as mining and fishing. As much as I love this game I do not have any time left over to play it and therefore I created a fully automated bot instead. What this bot dose is take a screenshot of game client and look for a pixel with a ceratin color (RGB-value) and then return the x- and y-coordinate. After fidning the coordinates a method is called to move the mouse cursor and click on the pixel with the specified coordinates.

I have used objected oriented programing (OOP) to be able to easily structure my code and make it readable. OOP also allows me to easily create multipule instances of the bot and run them at the same time by using threading. Since there will be multipule instances of this bot using the mouse cursor, I also had to create a method using mutual exclusion which only allows one bot at the time to use the mouse cursor.
